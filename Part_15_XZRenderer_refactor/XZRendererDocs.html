XZRenderer

A Vulkan-based 3D rendering engine built on SDL3 and ImGui. Include one header, write no Vulkan.

Overview

XZRenderer wraps Vulkan, SDL3, ImGui, Assimp and stb_image behind a clean C++ API. Your application only ever includes XZRenderer.hpp — no Vulkan or SDL headers leak into your code.

  • Blinn-Phong lighting with a single PointLight
  • glTF mesh loading via Assimp
  • Raw vertex / index buffer meshes
  • Custom fragment shader overlays (CustomShaderQuad)
  • Built-in ImGui widgets for transforms, lights and camera
  • SDL3 callback architecture — no manual event loop

Quick Start

The minimum application structure using SDL3 callbacks:

// main.cpp — only include XZRenderer.hpp
#define SDL_MAIN_USE_CALLBACKS 1
#include <SDL3/SDL_main.h>
#include <SDL3/SDL_main_impl.h>
#include "XZRenderer.hpp"

struct App {
    XZRenderer::Renderer    renderer;
    XZRenderer::MeshObject* sword = nullptr;
    XZRenderer::PointLight* light = nullptr;

    App() : renderer(800, 600, "My App") {}
};

SDL_AppResult SDL_AppInit(void** appstate, int argc, char* argv[])
{
    App* app = new App();
    *appstate = app;

    app->renderer.setClearColor(0.1f, 0.1f, 0.1f);
    app->renderer.setCameraPosition(0.0f, 0.0f, -6.0f);
    app->renderer.init();

    app->sword = &app->renderer.createMeshObject();
    app->sword->loadFromGLTF("assets/sword.glb", "assets/sword.jpg");
    app->sword->setPosition(0.0f, 0.0f, 0.0f);

    app->light = &app->renderer.createPointLight();
    app->light->setPosition(2.0f, 2.0f, -2.0f);

    return SDL_APP_CONTINUE;
}

SDL_AppResult SDL_AppEvent(void* appstate, SDL_Event* event)
{
    App* app = static_cast<App*>(appstate);
    if (app->renderer.handleEvent(event)) return SDL_APP_SUCCESS;
    return SDL_APP_CONTINUE;
}

SDL_AppResult SDL_AppIterate(void* appstate)
{
    App* app = static_cast<App*>(appstate);
    if (!app->renderer.beginFrame()) return SDL_APP_SUCCESS;

    XZRenderer::ImGuiLayer& gui = app->renderer.getGui();
    gui.beginWindow("Controls");
    gui.exposeTransformation(*app->sword, "Sword");
    gui.exposeLight(*app->light, "Light");
    gui.showFPS();
    gui.endWindow();

    app->renderer.endFrame();
    return SDL_APP_CONTINUE;
}

void SDL_AppQuit(void* appstate, SDL_AppResult result)
{
    delete static_cast<App*>(appstate);
}

Frame Loop

SDL3 drives the main loop via callbacks. XZRenderer fits into this naturally:

// SDL calls these in order every frame:
SDL_AppEvent    → forward to renderer.handleEvent()
SDL_AppIterate  → beginFrame() ... ImGui calls ... endFrame()
Important: All ImGuiLayer calls must happen between beginFrame() and endFrame().

beginFrame()

Acquires the next swapchain image, updates all uniform buffers (MVP matrices, light position, camera), and begins the ImGui frame. Returns false when the window is closed.

endFrame()

Renders ImGui, submits all command buffers to the GPU, and presents the frame.

Renderer

class Renderer // namespace XZRenderer

The top-level engine entry point. One instance per application. Non-copyable, non-movable.

Constructor

explicit Renderer(uint32_t width, uint32_t height, const std::string& title);

Width, height, and window title are required at construction time.

// In your App struct:
App() : renderer(800, 600, "My Window") {}

MeshObject

class MeshObject // namespace XZRenderer

A textured 3D object rendered with Blinn-Phong lighting. Created via Renderer::createMeshObject().

Loading

loadFromGLTF(mesh_path, texture_path)

Load geometry from a .glb file and a separate texture. Normals are generated automatically via Assimp.

app->sword->loadFromGLTF(
    "assets/sword_0.glb",
    "assets/sword_0.jpg"
);

loadFromVertices(vertices, indices, texture_path)

Load geometry from raw vertex and index data. Use XZRenderer::Vertex to supply positions, UVs and normals.

app->tetra->loadFromVertices(
    std::vector<XZRenderer::Vertex>(vertices, vertices + 12),
    std::vector<uint32_t>(indices, indices + 12),
    "assets/texture.png"
);

CustomShaderQuad

class CustomShaderQuad // namespace XZRenderer

A flat quad rendered with a user-supplied compiled fragment shader (.spv). The shared vertex shader is used automatically. Useful for procedural overlays — faces, decals, effects.

Creation

app->face = &app->renderer.createCustomShaderQuad("shaders/face.spv");
app->face->setVertices(positions, uvs, indices);
app->face->setPosition(-2.0f, 0.0f, 0.0f);

Methods

MethodDescription
setVertices(positions, uvs, indices)Supply quad geometry. Positions and UVs must be the same length.
setPosition(x, y, z)World position
setRotation(x, y, z)Rotation in degrees
setScale(x, y, z)Per-axis scale
Fragment shader: The quad uses alpha blending. Return alpha = 0 for transparent areas. The vertex shader outputs outUV at location 0.

PointLight

class PointLight // namespace XZRenderer

A positional light in the scene. Affects all MeshObject instances via Blinn-Phong lighting. Only the first light is currently used.

Methods

Shaders

XZRenderer expects compiled SPIR-V shaders in the build output directory.

FileUsed byDescription
vert.spvMeshObjectVertex shader — position, uv, normal. Outputs world position and transformed normal.
frag.spvMeshObjectFragment shader — Blinn-Phong lighting with texture sampling.
quad.spvCustomShaderQuadVertex shader — position and uv only. No normal.
face.spv (custom)CustomShaderQuadUser-supplied fragment shader. Receives outUV at location 0.

Compiling Shaders

glslangValidator -V -S vert shader.vert -o vert.spv
glslangValidator -V -S frag shader.frag -o frag.spv
glslangValidator -V -S vert quad.vert   -o quad.spv
glslangValidator -V -S frag face.frag   -o face.spv

Assets

Asset paths are passed directly to the API. Use ASSET_OUTPUT_DIR (defined by CMake) to prefix paths relative to the build output:

app->sword->loadFromGLTF(
    ASSET_OUTPUT_DIR "sword_0.glb",
    ASSET_OUTPUT_DIR "sword_0.jpg"
);
